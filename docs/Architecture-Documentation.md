# TomasAI Investment Fund Manager - Architecture Documentation

## Executive Summary

This document describes the current architecture of the TomasAI Investment Fund Manager (IFM) system, a sophisticated financial trading platform built on **.NET 8** using an **Event Model Actor-Based Architecture**. The system implements Command Query Responsibility Segregation (CQRS), Event Sourcing, and the Actor Model patterns to achieve high scalability, fault tolerance, and eventual consistency.

## 3. Event Model Actor System

### Core Actor Concepts

The system implements a mailbox-based actor model where each actor:
- Has a unique **ActorMailboxId** (ActorType + Name)
- Processes messages sequentially via its mailbox
- Maintains isolated state per **ActorThreadId** (ActorType + Name + EntityId)
- Communicates via asynchronous message passing

### Actor Identification

```csharp
// Mailbox identifies the actor type (one per actor class)
ActorMailboxId = (ActorType, Name)
// Example: (Command, "FuturesContractCommand")

// Thread identifies a specific entity instance
ActorThreadId = (ActorType, Name, EntityId)
// Example: (Command, "FuturesContractCommand", "ES-2025-06")

// Subject routes messages to specific verbs
ActorSubject = (ActorType, Name, Verb, EntityId)
// Example: (Command, "FuturesContractCommand", "Add", "ES-2025-06")
```

### Repository Pattern

```csharp
// Base Event Source Repository
public abstract class BaseEventSourceActorRepository
{
    // Load state by replaying events
    protected async Task<TState> LoadStateAsync<TState>(ICommand command)
        where TState : IEventSourceActorState<TState>
    {
        var streamId = await GetStreamId(command.StreamId);
        var state = _stateFactory.CreateState<TState>();
        await _dbEventSource.MapReduceActorEventStreamAsync<TState>(
            streamId, 
            e => state.ReplayEvents(e.Select(o => o.ToDomainEvent()))
        );
        return state;
    }
    
    // Save state by persisting events and publishing
    protected async Task SaveStateAsync<TState>(TState state, ICommand command)
        where TState : IEventSourceActorState<TState>
    {
        foreach (var domainEvent in state.Events)
        {
            await _dbEventSource.InsertEventAsync(streamId, domainEvent);
            await _actorService.SendAsync(domainEvent);  // Publish to denormalizer
        }
    }
}
```
